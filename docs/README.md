# Documentation

This directory is organized by audience and job-to-be-done rather than by creation date.

## Structure

- `architecture/` - System design, runtime behavior, and technical reference material.
- `operations/` - Runbooks, environment setup, and maintenance procedures.
- `planning/` - Backlogs, launch planning, deferred work, and the append-only major changes log.
- `editorial/` - Station curation, taxonomy, and editorial data-entry guidance.
- `security/` - Threat modeling and security-specific reference documents.

## Key documents

- `architecture/asset-architecture.md` - Media asset storage and delivery design.
- `architecture/reliability-and-metadata.md` - Stream health, metadata routing, and now-playing behavior.
- `architecture/stream-probing.md` - Stream probe behavior, timings, and persistence rules.
- `operations/editorial-sync.md` - Local to staging or production editorial sync workflow.
- `operations/db-maintenance.md` - Live database access and maintenance queries.
- `planning/major-changes.md` - Append-only log of major project changes.
- `planning/launch-readiness.md` - Public launch checklist and gating items.
- `planning/pending-security-issues.md` - Security backlog that remains open.
- `editorial/station-entry-guide.md` - Editorial checklist for adding or updating stations.
- `editorial/station-tagging-strategy.md` - Taxonomy rules and tagging constraints.
- `editorial/tag-system-reference.md` - Quick-reference version of the controlled tag dimensions.
- `security/threat-model.md` - OWASP-oriented trust-boundary and risk map.
